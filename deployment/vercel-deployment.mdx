## Overview

Cajas is optimized for deployment on Vercel, leveraging Next.js 16's advanced features including React Compiler and server-side rendering. This guide walks through the complete deployment process from initial setup to custom domain configuration.

## Pre-Deployment Checklist

Before deploying to production, ensure you have:

<Steps>
  <Step title="Supabase Project">
    - Production Supabase project created
    - All migrations applied
    - RLS policies enabled and tested
    - Auth providers configured
  </Step>
  
  <Step title="Environment Variables">
    - Production API keys secured
    - Database connection strings ready
    - Third-party service credentials prepared
  </Step>
  
  <Step title="Code Quality">
    - Build passes locally (`npm run build`)
    - No TypeScript errors
    - Lint checks pass (`npm run lint`)
  </Step>
</Steps>

## Initial Deployment

### 1. Connect to Vercel

<CodeGroup>
```bash Vercel CLI
# Install Vercel CLI
npm i -g vercel

# Login to your account
vercel login

# Deploy from project directory
cd workspace/source
vercel
```

```bash GitHub Integration
# Push your code to GitHub
git remote add origin https://github.com/yourusername/cajas.git
git push -u origin main

# Then import on Vercel dashboard:
# 1. Visit https://vercel.com/new
# 2. Select your repository
# 3. Configure project settings
```
</CodeGroup>

### 2. Configure Build Settings

Vercel automatically detects Next.js projects. Verify these settings:

<AccordionGroup>
  <Accordion title="Build Configuration">
    - **Framework Preset**: Next.js
    - **Build Command**: `next build`
    - **Output Directory**: `.next`
    - **Install Command**: `npm install`
    - **Node Version**: 20.x (recommended)
  </Accordion>
  
  <Accordion title="Advanced Settings">
    ```json next.config.ts
    {
      "reactCompiler": true,
      "productionBrowserSourceMaps": false,
      "webpack": {
        "ignoreWarnings": [
          { "module": "/node_modules/" },
          { "message": "/sourceMapURL/" }
        ]
      }
    }
    ```
    
    The configuration in `next.config.ts:3-14` enables:
    - React Compiler for optimized rendering
    - Disabled source maps for security
    - Webpack warning suppression
  </Accordion>
</AccordionGroup>

## Environment Variables

### Required Variables

Configure these environment variables in your Vercel project settings:

<ParamField path="NEXT_PUBLIC_SUPABASE_URL" type="string" required>
  Your Supabase project URL
  
  ```bash
  https://your-project.supabase.co
  ```
  
  Used in: `lib/supabase/server.ts:8`, `lib/supabase/client.ts:5`
</ParamField>

<ParamField path="NEXT_PUBLIC_SUPABASE_ANON_KEY" type="string" required>
  Supabase anonymous (public) key for client-side requests
  
  ```bash
  eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
  ```
  
  Used in: `lib/supabase/server.ts:9`, `lib/supabase/client.ts:6`
</ParamField>

<ParamField path="SUPABASE_SERVICE_ROLE_KEY" type="string" required>
  Service role key for server-side admin operations (keep secret)
  
  <Warning>
    Never expose this key in client-side code. Only use in API routes and server components.
  </Warning>
</ParamField>

### Setting Variables on Vercel

<Tabs>
  <Tab title="Dashboard">
    1. Navigate to your project on Vercel
    2. Go to **Settings** → **Environment Variables**
    3. Add each variable with appropriate scope:
       - **Production**: Live environment
       - **Preview**: Pull request deployments
       - **Development**: Local development
    4. Click **Save**
  </Tab>
  
  <Tab title="CLI">
    ```bash
    # Add production variable
    vercel env add NEXT_PUBLIC_SUPABASE_URL production
    
    # Add to all environments
    vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY
    
    # Pull variables to local .env
    vercel env pull
    ```
  </Tab>
  
  <Tab title="Bulk Import">
    Create a `.env.production` file:
    
    ```bash .env.production
    NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
    SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
    ```
    
    Then import via CLI:
    ```bash
    vercel env import .env.production
    ```
  </Tab>
</Tabs>

## Custom Domains

### Adding a Domain

<Steps>
  <Step title="Add Domain in Vercel">
    1. Go to **Settings** → **Domains**
    2. Enter your domain (e.g., `cajas.club`)
    3. Click **Add**
  </Step>
  
  <Step title="Configure DNS">
    Add these DNS records at your domain provider:
    
    <CodeGroup>
    ```txt A Record (Apex Domain)
    Type: A
    Name: @
    Value: 76.76.21.21
    TTL: 3600
    ```
    
    ```txt CNAME Record (www)
    Type: CNAME
    Name: www
    Value: cname.vercel-dns.com
    TTL: 3600
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Verify Configuration">
    Vercel will automatically verify DNS propagation. This typically takes 5-10 minutes but can take up to 48 hours.
    
    Check status:
    ```bash
    vercel domains inspect cajas.club
    ```
  </Step>
  
  <Step title="Configure SSL">
    SSL certificates are automatically provisioned and renewed by Vercel using Let's Encrypt. No additional configuration needed.
  </Step>
</Steps>

## Production Optimizations

### Performance Configuration

```json vercel.json
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "X-Content-Type-Options",
          "value": "nosniff"
        },
        {
          "key": "X-Frame-Options",
          "value": "DENY"
        },
        {
          "key": "X-XSS-Protection",
          "value": "1; mode=block"
        },
        {
          "key": "Strict-Transport-Security",
          "value": "max-age=31536000; includeSubDomains"
        }
      ]
    },
    {
      "source": "/static/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=31536000, immutable"
        }
      ]
    }
  ],
  "rewrites": [
    {
      "source": "/api/:path*",
      "destination": "/api/:path*"
    }
  ]
}
```

### Next.js Configuration

The production build leverages several optimizations:

<AccordionGroup>
  <Accordion title="React Compiler" icon="react">
    Enabled via `reactCompiler: true` in `next.config.ts:5`
    
    Benefits:
    - Automatic memoization
    - Reduced bundle size
    - Improved runtime performance
    - Better code splitting
  </Accordion>
  
  <Accordion title="Source Maps" icon="map">
    Disabled via `productionBrowserSourceMaps: false` in `next.config.ts:6`
    
    Security benefits:
    - Prevents source code exposure
    - Reduces bundle size
    - Faster page loads
  </Accordion>
  
  <Accordion title="Webpack Optimizations" icon="box">
    Configured in `next.config.ts:7-13`
    
    ```typescript
    webpack: (config) => {
      config.ignoreWarnings = [
        { module: /node_modules/ },
        { message: /sourceMapURL/ }
      ];
      return config;
    }
    ```
  </Accordion>
</AccordionGroup>

## Deployment Workflow

### Automatic Deployments

Vercel provides automatic deployments for every push:

<Steps>
  <Step title="Push to Branch">
    ```bash
    git checkout -b feature/new-feature
    git add .
    git commit -m "Add new feature"
    git push origin feature/new-feature
    ```
  </Step>
  
  <Step title="Preview Deployment">
    Vercel automatically creates a preview deployment with a unique URL:
    ```
    https://cajas-{hash}-yourteam.vercel.app
    ```
    
    Preview deployments include:
    - Full production build
    - Preview environment variables
    - Unique URL for testing
    - Comment on PR with deployment link
  </Step>
  
  <Step title="Production Deployment">
    Merge to `main` branch triggers production deployment:
    ```bash
    git checkout main
    git merge feature/new-feature
    git push origin main
    ```
    
    Production deployment:
    - Updates live site
    - Invalidates CDN cache
    - Runs on custom domain
  </Step>
</Steps>

### Manual Deployments

<CodeGroup>
```bash Redeploy Current
# Redeploy latest commit
vercel --prod
```

```bash Deploy Specific Commit
# Deploy a specific commit
git checkout abc123
vercel --prod
```

```bash Rollback
# List deployments
vercel ls

# Promote previous deployment to production
vercel promote [deployment-url] --scope=your-team
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build Failures" icon="triangle-exclamation">
    **Symptom**: Deployment fails during build step
    
    **Solutions**:
    1. Check build logs in Vercel dashboard
    2. Verify environment variables are set
    3. Test build locally:
       ```bash
       npm run build
       npm start
       ```
    4. Check for TypeScript errors
    5. Verify all dependencies are in `package.json`
  </Accordion>
  
  <Accordion title="Runtime Errors" icon="bug">
    **Symptom**: Build succeeds but application crashes at runtime
    
    **Solutions**:
    1. Check function logs in Vercel dashboard
    2. Verify Supabase connection:
       ```typescript
       // Test in API route
       const { data, error } = await supabase.from('cases').select('count');
       console.log({ data, error });
       ```
    3. Check CORS configuration
    4. Verify environment variables are accessible
  </Accordion>
  
  <Accordion title="Database Connection Issues" icon="database">
    **Symptom**: Cannot connect to Supabase
    
    **Solutions**:
    1. Verify `NEXT_PUBLIC_SUPABASE_URL` is correct
    2. Check Supabase project is active
    3. Verify API keys are not expired
    4. Test connection in Supabase dashboard
    5. Check network/firewall settings
  </Accordion>
  
  <Accordion title="Domain Not Working" icon="globe">
    **Symptom**: Custom domain shows error or doesn't load
    
    **Solutions**:
    1. Verify DNS propagation:
       ```bash
       dig cajas.club
       nslookup cajas.club
       ```
    2. Check DNS records match Vercel requirements
    3. Wait for DNS propagation (up to 48 hours)
    4. Verify domain is not already in use
    5. Check domain registrar settings
  </Accordion>
</AccordionGroup>
